== Session 1 – Mind Map Verbesserungen

=== Prompt 1

[quote, ryanj]
____
Verbesserungen:
Nodes dürfen sich nicht überlappen.
Nutze all den Platz den der Bildschirm hergibt (nicht nur nach rechts anfügen).
Erstelle Keyboard shortcuts um an die „aktuelle" Node eine Node anzuhängen, auch um zwischen verschiedenen Nodes zu wechseln um sie „aktuell" zu machen.
Mach visuell klar welche node aktuell ist und welche die Ursprungsnode ist.
____

*Umgesetzte Änderungen:*

* Radiales Layout-Algorithmus: Root-Node in der Mitte, Kinder werden in alle Richtungen (360°) verteilt, gewichtet nach Teilbaumgröße → keine Überlappungen
* Keyboard-Shortcuts:

[cols="1,2", options="header"]
|===
| Taste | Aktion
| `Enter` / `Ins` | Kind-Node zur aktuellen Node hinzufügen
| `↑` / `↓` | Zwischen Geschwistern navigieren
| `→` | Zum ersten Kind navigieren
| `←` | Zum Elternteil navigieren
| `Tab` / `Shift+Tab` | Alle Nodes der Reihe nach durchlaufen
| `F2` | Text der aktuellen Node bearbeiten
| `Del` | Aktuelle Node löschen
|===

* Visuelle Hervorhebung:
** Root-Node: goldgelb (`#FFD700`)
** Aktuelle (selektierte) Node: blauer Rand (`DodgerBlue`, 3.5px)
* Klick auf Node macht sie zur aktuellen

=== Prompt 2

[quote, ryanj]
____
es kompiliert nicht du nga
____

*Problem:* Java 16+ Syntax in einem Java 11 Projekt verwendet:

* `instanceof MindMap map` (Pattern Matching → erst ab Java 16)
* `case ENTER, INSERT ->` (Enhanced Switch Arrow → erst ab Java 14)
* Fehlendes `import javafx.scene.input.KeyCode`

*Fix:* Syntax auf Java 11 kompatible Varianten umgeschrieben (`instanceof` mit Cast, `if/else if` statt Switch-Expression, Import ergänzt).

*Ursachen und Fixes:*

[cols="1,2", options="header"]
|===
| Problem | Lösung

| `MindMap.removeNode()` fehlte | Methode hinzugefügt
|===
